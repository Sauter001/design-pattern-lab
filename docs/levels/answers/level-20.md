# 레벨 20 답 맞추기: 정답과 이정표

스포일러 주의. **after 커밋이랑 회고까지 끝낸 다음에만 읽어요.** 아직이면 [레벨 20으로 돌아가요](../level-20.md). 이번 건 레벨 1, 7 정답이랑 같이 펴면 더 빨리 보여요.

## 다시 발견한 것

커맨드 패턴(Command)이에요.

요청 실행을 `execute()` 하나 가진 객체로 떴죠. 핸들러랑 인자랑 문맥까지 그 안에 품게요. 그래서 받은 시점과 실행 시점이 갈라졌고요. 즉시 실행해도 되고, 큐에 넣었다 나중에 해도 되고, 실패하면 같은 객체를 다시 불러도 되죠. 실행이라는 행위 자체를 1급 객체로 떠서 들고 다닐 수 있게 된 것, 그게 커맨드예요.

## 카탈로그와 대조하기

refactoring.guru: [커맨드](https://refactoring.guru/ko/design-patterns/command)

맞춰 볼 것:

- **Command**(공통 약속, `execute`) = 본인의 실행 객체, **ConcreteCommand** = 특정 요청 한 건의 실행(핸들러와 인자를 품은), **Invoker** = 큐/워커/디스패처(`execute`만 부르는 쪽), **Receiver** = 실제 일하는 핸들러, **Client** = 커맨드를 만들어 Invoker에 건네는 dispatch.
- 카탈로그가 강조하는 한 줄. "Invoker는 Receiver를 직접 모르고 Command만 안다." 그래서 실행을 큐에 넣고, 로깅하고, 미룰 수 있어요. 본인 코드에서 큐랑 워커가 무슨 요청인지 모른 채 `execute()`만 부르는지 보세요. 그러면 제대로 잡은 거예요.
- **되돌리기(undo).** GoF 커맨드의 간판 기능이에요. `execute`랑 짝으로 `undo`를 두면 실행 취소 스택이 되죠. 오늘은 큐잉이랑 재시도로 갔지만, undo가 같은 패턴의 또 다른 얼굴이라는 것만 알아 두세요. 떠 둔 객체라서 되돌리기도 얹을 수 있는 거예요.

## 전략(레벨 1, 7)과 나란히 두기

둘 다 "할 일을 객체로 뜬다"는 점이 닮아서 헷갈려요. 그래서 갈라 둘게요.

- **전략**(레벨 1, 7)은 "어떻게 할지(알고리즘)"를 갈아끼워 **지금** 실행해요. 보통 상태가 없는 방법 그 자체고요.
- **커맨드**(레벨 20)는 "무엇을 할지(요청 한 건)"를 인자까지 담아 떠서, **언제** 실행할지를 미뤄요. 인자를 품은 실행 한 건이라, 큐에 쌓였다 나중에 불리고 실패하면 다시 불리죠.

한 문장으로. 전략은 "어떻게", 커맨드는 "무엇을, 언제". 회고에 이 한 줄 적어 두면 다신 안 헷갈려요.

## 비용과 안 쓸 자리 (꼭)

커맨드도 공짜가 아니에요. 요청 종류가 많으면 클래스가 우수수 늘죠(람다로 완화는 돼요). 그리고 "받아서 바로 실행"이 두 단계로 갈라져서, 흐름 좇기가 한 겹 어려워지고요. **즉시 실행만 하면 이건 과해요.** 미루기(큐), 다시 하기(재시도), 되돌리기(취소), 기록(감사) 중 하나라도 정말 필요할 때라야 값을 해요. 본인 요청이 그중 뭘 필요로 했는지 적었으면, 그게 이 패턴을 쓸지 말지 가르는 기준이에요.

회고 빈칸: "내 요청은 미루기/재시도/되돌리기/기록 중 ___가 필요했다. 그래서 커맨드가 ___했다(값을 했다 / 과했다)."
