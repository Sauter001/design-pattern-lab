# 도메인 토대: 레벨에 들어가기 전에

레벨 0 읽고 왔죠. 본론 들어가기 전에 바닥 한 번 깔고 가요. 안 그러면 레벨 7쯤 가서 `Row`가 어디서 튀어나왔냐고, `Table.rows()`는 누가 만든 거냐고 헤매거든요. 매번 봐요. 그래서 미리 정리해 둬요. 길지 않아요.

## 먼저, 헷갈리는 것부터 정리

이 lab은 **레벨마다 따로 노는 26개 예제가 아니에요.** 프로젝트 두 개(미니 DB, 미니 스프링)를 레벨에 걸쳐 **한 덩어리로 키워 가는** 거예요. 레벨 7에서 만든 엔진 조각 위에 레벨 11이 얹히고, 레벨 19가 또 얹히는 식으로요. 그러니 레벨마다 본 before 코드는 허공에서 시작하는 게 아니라, **공용 바닥 위에 서 있어요.** 그 바닥이 이 문서예요.

레벨 본문은 패턴 자체에 집중하느라 이 바닥을 매번 다시 설명 안 해요. 그래서 여기 한곳에 모아 둡니다. 레벨 가다가 "이 타입 어디서 났지?" 싶으면 여기로 돌아오면 돼요.

## 두 종류를 가릅시다 (이게 핵심)

이 lab에 나오는 타입은 두 부류예요. 섞으면 안 돼요.

1. **토대 타입 (운반체).** `Row`, `Table`, `Page`, `Request` 같은 거요. 그냥 데이터를 담아 나르는 그릇이에요. 패턴이랑 아무 상관 없어요. 모양 고민할 거리도 아니고요. 그래서 **제가 미리 깔아 두거나, 여기서 복사해 쓰라고 줍니다.** 이건 발견할 게 없어요.

2. **구조 타입 (패턴 주제).** `Query`, `Plan`, `Condition`(조건 트리), 빈 팩토리, WAL 같은 거요. 이건 **본인이 해당 레벨에서 직접 짓는 거예요.** 바로 이걸 어떻게 조직하느냐가 그 레벨의 패턴이거든요. 그러니 여기서 미리 안 줘요. 주면 김새요.

한 줄로. **그릇은 드릴게요. 그릇에 뭘 어떻게 담을지가 레벨이에요.**

## 미니 DB 토대

모듈 셋(`storage` <- `engine` <- `frontend`), 의존은 아래에서 위로예요. 아래 토대 타입은 **이미 소스로 깔려 있어요.** 그냥 import해서 쓰면 돼요.

### `lab.db.engine.Row` (제공됨)

행 하나. 컬럼 이름에서 값으로 가는 작은 지도예요. 레벨 7 조인의 `l.get(key)`, 레벨 22 평가의 `evaluate(Row)`가 이걸 씁니다.

    Row r = Row.of("id", 1, "name", "kim", "age", 20);
    r.get("age");                 // 20
    r.merged(otherRow);           // 두 행 합치기 (조인에서 편함)

### `lab.db.engine.Table` (제공됨)

이름 붙은 테이블. 행들의 모음이에요. 레벨 7이 `table.rows()`로 훑어요.

    Table users = new Table("users")
        .add(Row.of("id", 1, "name", "kim"))
        .add(Row.of("id", 2, "name", "lee"));
    users.rows();                 // List<Row>

### `lab.db.storage.Page` (제공됨)

저장 페이지 하나. 바이트 덩어리예요. 레벨 8 트랜잭션, 레벨 13 롤백 스냅샷이 이 위에서 굴러가요.

    Page p = new Page(1);
    p.write(bytes);
    byte[] snapshot = p.read();   // 복사본 (레벨 13에서 쓸모 있음)

### 본인이 짓는 것 (미리 안 줌)

- `Query` / `Plan`: 레벨 11(쿼리 조립)에서 짓고, 레벨 19(최적화), 24(실행)가 다듬고 굴려요.
- `Condition` 트리(`Comparison`, `BoolGroup` 등): 레벨 21에서 짓고 22, 23이 재사용해요.
- 조인 알고리즘, 옵티마이저 규칙, 실행 연산자: 각 레벨에서요.
- 레벨 8 before의 `log`(WAL): 그 전까진 최소 스텁으로 두세요. 레벨 14에서 제대로 만들어요.

## 미니 스프링 토대

모듈 둘(`container` <- `mvc`)이에요. 여긴 토대를 미리 소스로 안 깔았어요. 미니 DB만큼 공유 타입이 많지 않아서요. 대신 필요해지는 레벨에서 아래 최소형을 복사해 넣으면 돼요. 필드는 본인 흐름에 맞춰 늘려도 되고요.

### `Handler` (레벨 1에서 본인이 만듦)

이건 토대로 안 줘요. 레벨 1이 이걸 만드는 게 첫 실습이거든요. 처음엔 경로 문자열을 받지만(`String handle(String path)`), 레벨 20쯤 요청 객체를 받게 진화해요. 그 진화도 레벨이 안내해요.

### `Request` / `Response` (레벨 20 즈음 복사)

    public record Request(String path, Map<String, String> params) {}
    public record Response(int status, String body) {}

### `BeanDefinition` (레벨 9, 10 즈음 복사)

빈을 "어떻게 만들지" 적어 둔 쪽지예요. 빈 인스턴스 자체가 아니라요. 빈을 만드는 구조(팩토리)는 레벨 9, 10에서 본인이 짓고, 이건 그 입력이에요.

    public record BeanDefinition(String name, Class<?> type, String scope) {}

## 정리

- 레벨은 한 시스템을 키워 가는 연재예요. before는 공용 바닥 위에 서요.
- 그릇(`Row`, `Table`, `Page` 등)은 제공. 구조(`Query`, `Condition`, 팩토리 등)는 본인 몫.
- 바닥 빌드 확인은 `./gradlew build` 하면 돼요. 토대 타입은 이미 컴파일되니, 초록불 한 번 보고 [레벨 1](level-01.md)로 가요.

자, 바닥 깔았어요. 이제 진짜 시작이에요.
