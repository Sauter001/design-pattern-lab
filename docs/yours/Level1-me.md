# 레벨 1 패턴 단위 회고

# [패턴명] - [등장 도메인]

## 문제 상황
어떤 도메인에서 무슨 변경 압력이 반복되는가.

## before의 고통
패턴 없는 버전에서 [변경 요구사항]이 들어오면 구체적으로 어디를 몇 군데 고쳐야 하는가.

## 적용한 패턴과 의도
무엇을, 왜. (Proxy인가 Decorator인가, 의도로 구분)

## after의 개선
같은 변경이 이제 어디만 닿는가.

## 대안 패턴
다른 패턴으로도 풀 수 있었는가. 왜 이걸 골랐는가. (혹은 왜 안 썼는가)

## 트레이드오프 (비용)
이 패턴이 추가로 떠안긴 비용: 클래스 증가, 간접성, 디버깅 난이도.

## [해당 시] 의도 삽입 고지
도메인이 요구해서가 아니라 학습 목적상 심은 패턴이면 명시.

## 관련 커밋
before -> 고통 -> after 커밋 해시를 적어 서사를 연결한다.
